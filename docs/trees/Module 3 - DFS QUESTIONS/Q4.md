# Mirror a Binary Tree

Given:

         1
       /   \
      2     3
     / \
    4   5

Transform it into:

        1
      /   \
     3     2
          / \
         5   4



Notice something...

We're not returning an int.

We're not returning a boolean.

We're changing the tree itself.

This introduces a completely different style of recursion.

# Answer

1. For this problem, each recursive call is asking:

"Mirror this subtree."

Notice the difference.

The function isn't asking for information anymore.

It is asking the subtree to perform an action.

This is our first recursive modification algorithm.

2. Unlike previous lessons, this method doesn't return an int.

It probably looks like this:

public static void mirror(TreeNode node)

The return type is now:

void

because we're modifying the existing tree.

3. So what gets swapped.

For example:

values?
objects?
references?


suppose we have:

    TreeNode node = ...;

and

      8
     / \
    3   10

In memory, it looks like this:

        node
        │
        ▼
        +----------------------+
        | value = 8            |
        | left -----> (3)      |
        | right ----> (10)     |
        +----------------------+

When we mirror the tree, we are not moving the node with value 3.

We are not moving the node with value 10.

Those objects stay exactly where they are in memory.

We only change this:

node.left
node.right

After swapping:

    node
    │
    ▼
    +----------------------+
    | value = 8            |
    | left -----> (10)     |
    | right ----> (3)      |
    +----------------------+

The objects never moved.

Only the references changed.


4.  Suppose we start with:

           1
         /   \
        2     3
             / \
            4   5
   
## Option A

Swap first.

            1
          /   \
         3     2
              / \
             4   5

Now recurse.

Mirror subtree 3.

Nothing changes.

Mirror subtree 2.
    
          2
         / \
        5   4

Final result:

            1
          /   \
         3     2
              / \
             5   4

Correct.

Now try Option B.

Don't swap yet.

First recurse on the left subtree.

Subtree 2 becomes:

      2
     / \
    5   4

The tree is now:

         1
       /   \
      2     3
     / \
    5   4

Now recurse on 3.

Nothing changes.

Finally...

Swap the children of the root.

What do we get?

        1
      /   \
     3     2
          / \
         5   4

Wait.

That's exactly the same tree.

So in this particular algorithm:

Swap then recurse 
Recurse then swap 

Both work.
